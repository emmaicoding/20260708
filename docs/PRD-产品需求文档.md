# 幼儿园管理系统 — 产品需求文档（PRD）

> **文档版本：** v1.0
> **创建日期：** 2026-07-06
> **文档性质：** 课程大作业 — 产品需求规格说明

---

## 1. 项目背景

幼儿园设有大班、中班、小班各3个班级，每班10名儿童，共90名儿童。幼儿园提供各类课程和兴趣班。目前缺乏信息化管理手段，需要开发一套**Java控制台应用程序**，实现幼儿信息、班级、课程、食谱、考勤等核心业务的数字化管理。

## 2. 项目目标

| 目标维度 | 具体描述 |
|---------|---------|
| 功能目标 | 实现幼儿学籍、食谱、课程、调班、考勤、统计6大核心模块 |
| 技术目标 | Java + MySQL + 分层架构（View → Service → DAO → Entity） |
| 质量目标 | 代码注释完整，每个类标明编写者、完成时间、功能说明 |
| 交付目标 | 可运行的控制台程序 + PPT + 项目文档 |

## 3. 用户角色定义

### 3.1 管理员（Admin）
- 拥有系统全部权限
- 管理幼儿信息（增删改查）
- 管理班级信息
- 管理课程信息（添加/删除课程，设置课程容量上限）
- 管理食谱（菜品库 + 每周排餐）
- 执行调班操作
- 查看数据统计报表

### 3.2 教师（Teacher）
- 查看本班幼儿信息
- 记录每日考勤（出勤/缺勤/请假）
- 查看本班课程安排
- 查看本周食谱
- 查看个人负责的课程及学员名单

## 4. 功能需求详述

### 4.1 登录与权限模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-010 | 用户登录 | 输入用户名+密码，验证身份后进入系统 | P0 |
| F-011 | 菜单权限区分 | 根据角色展示不同的功能菜单 | P0 |
| F-012 | 修改密码 | 用户可修改自己的登录密码 | P1 |

### 4.2 幼儿学籍管理模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-020 | 添加幼儿 | 录入姓名、性别、出生日期、家长联系方式、所在班级 | P0 |
| F-021 | 修改幼儿信息 | 修改幼儿基本信息（非班级字段） | P0 |
| F-022 | 删除幼儿 | 幼儿离园时删除学籍（同时清除选课、考勤记录） | P0 |
| F-023 | 查询幼儿详情 | 根据幼儿ID/姓名查询，展示基本信息+所在班级+所学课程 | P0 |
| F-024 | 按班级查看幼儿 | 选择某个班级，列出该班所有幼儿名单 | P0 |
| F-025 | 幼儿列表分页 | 幼儿数量较多时支持分页展示 | P1 |

**幼儿信息字段：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| name | VARCHAR(50) | 幼儿姓名 |
| gender | CHAR(1) | 性别：M/F |
| birth_date | DATE | 出生日期 |
| parent_name | VARCHAR(50) | 家长姓名 |
| parent_phone | VARCHAR(20) | 家长联系电话 |
| class_id | INT | 所在班级ID（外键） |
| enrollment_date | DATE | 入园日期 |
| status | TINYINT | 状态：1在园/0离园 |

### 4.3 班级管理模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-030 | 查看所有班级 | 展示大班/中班/小班各3个班级及人数统计 | P0 |
| F-031 | 查看班级详情 | 选择班级后查看：班级名称、年级、当前人数、幼儿名单 | P0 |

**班级数据结构（预置9个班级）：**

| 年级 | 班级名称 |
|------|---------|
| 大班 | 大一班、大二班、大三班 |
| 中班 | 中一班、中二班、中三班 |
| 小班 | 小一班、小二班、小三班 |

### 4.4 课程管理模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-040 | 查看所有课程 | 列出系统中所有兴趣课程及已选人数/容量上限 | P0 |
| F-041 | 添加课程 | 管理员新增一门兴趣课程（名称+容量上限） | P0 |
| F-042 | 修改课程信息 | 修改课程名称或容量上限 | P1 |
| F-043 | 删除课程 | 删除课程（需先清退已选该课的幼儿） | P1 |
| F-044 | 为幼儿选课 | 为某个幼儿选择兴趣课程（每人限选4门） | P0 |
| F-045 | 退课 | 为幼儿取消某门已选课程 | P0 |
| F-046 | 查看幼儿课程 | 查询某个幼儿所学的全部兴趣课程 | P0 |
| F-047 | 查看课程学员 | 查询某门课程的所有选课幼儿名单 | P0 |

**课程业务规则：**
- 每个幼儿**必须且只能**选择4门兴趣课程
- 每门课程有**容量上限**（如默认15人），达到上限后不可再选
- 预置课程：舞蹈、跆拳道、钢琴、美术（管理员可自行增删课程）

### 4.5 食谱管理模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-050 | 菜品库管理 | 添加/修改/删除菜品（名称、类型：主食/荤菜/素菜/汤/水果） | P0 |
| F-051 | 每周排餐 | 按周一~周五，为每天的早/中/晚餐分配菜品 | P0 |
| F-052 | 查看本周食谱 | 展示本周周一到周五的三餐安排 | P0 |
| F-053 | 查看某天食谱 | 选择某一天查看当日三餐详情 | P1 |
| F-054 | 复制上周食谱 | 将上周食谱复制为本周模板，再微调 | P2 |

### 4.6 调班管理模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-060 | 执行调班 | 选择幼儿 → 选择目标班级 → 确认调班 | P0 |
| F-061 | 调班记录 | 记录每次调班操作（操作人、时间、原班级、新班级） | P1 |
| F-062 | 调班校验 | 调班前校验：目标班级人数是否已满（默认每班上限10人） | P0 |

**调班业务规则：**
- 支持跨年级调班（如从小班调到中班）
- 调班后自动更新幼儿的class_id
- 记录调班历史日志

### 4.7 考勤管理模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-070 | 每日考勤打卡 | 教师为本班幼儿逐一记录出勤/缺勤/请假状态 | P0 |
| F-071 | 查看当日考勤 | 查看某班当日的考勤汇总 | P0 |
| F-072 | 查看历史考勤 | 按日期范围查询某个幼儿或某个班级的考勤记录 | P1 |
| F-073 | 考勤统计 | 统计某幼儿/某班在指定时间段的出勤率 | P1 |

**考勤状态枚举：** 出勤(1) / 缺勤(2) / 请假(3) / 迟到(4)

### 4.8 数据统计报表模块

| 编号 | 功能项 | 描述 | 优先级 |
|------|--------|------|--------|
| F-080 | 班级人数统计 | 各班级当前人数及男女比例 | P0 |
| F-081 | 课程选课统计 | 各课程已选人数/剩余容量，选课率排名 | P0 |
| F-082 | 年级分布统计 | 各年级总人数、班级平均人数 | P1 |
| F-083 | 出勤率报表 | 指定时间段内各班出勤率对比 | P1 |

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 架构 | 分层架构：View → Service → DAO → Entity，层间通过接口调用 |
| 数据库 | MySQL 8.x，使用JDBC连接 |
| 代码规范 | 每个类必须有完整注释：类说明、编写者、完成时间、版本号 |
| 输入校验 | 所有用户输入需做合法性校验（如日期格式、手机号格式、非空校验） |
| 异常处理 | 数据库操作异常需捕获并友好提示，禁止直接打印堆栈信息 |
| 初始化 | 系统首次运行时自动初始化数据库表结构和预置数据（9个班级、4门课程、管理员账号） |

## 6. 数据规模

| 数据实体 | 预估数量 |
|---------|---------|
| 幼儿 | 90人（每班10人） |
| 班级 | 9个（3年级 × 3班） |
| 兴趣课程 | 4~10门（动态可扩展） |
| 用户账号 | 5~10个（含管理员和教师） |
| 每周食谱 | 5天 × 3餐 = 15条/周 |
| 考勤记录 | 90人 × 22天/月 ≈ 1980条/月 |

## 8. 验收标准

1. 所有P0功能可通过控制台正常操作并持久化到MySQL
2. 系统启动时自动建表并插入预置数据（无需手动执行SQL脚本）
3. 每个选课操作校验课程容量，每个调班操作校验班级容量
4. 删除幼儿时级联清除相关选课和考勤记录
5. 代码100%有类级别注释，关键方法有功能注释
6. 分层架构清晰，View层不直接调用DAO层

# Bug 修复记录

## v1.0.1 — 2026-07-08

### Bug #1：幼儿出生日期与班级不匹配

| 字段 | 内容 |
|------|------|
| 问题描述 | 所有幼儿的出生日期随机分配（2020/2021/2022轮替），大班、中班、小班幼儿年龄混在一起，不符合实际幼儿园按年龄分班的逻辑 |
| 问题原因 | `InitDatabase.java` 第256行使用 `2020 + (childId % 3)` 按 childId 轮替分配出生年份，未根据班级年级区分 |
| 处理人 | 李瑞 |
| 修复方案 | 根据班级ID确定出生年份：大班(classId 1~3)→2020年，中班(classId 4~6)→2021年，小班(classId 7~9)→2022年 |
| 修复版本 | v1.0.1 |
| 修复文件 | `src/kindergarten/util/InitDatabase.java` |

### Bug #2：所有幼儿都选了全部4门课程

| 字段 | 内容 |
|------|------|
| 问题描述 | 90名幼儿每人选了全部4门兴趣课程，每门课都是90人，无法体现"选课"的差异性和容量校验功能 |
| 问题原因 | `InitDatabase.java` 第276~281行使用双重循环为每个幼儿全量插入4门课程，没有任何随机性 |
| 处理人 | 李瑞 |
| 修复方案 | 改为根据childId轮替分配2~4门课程（`2 + (cid % 3)`），课程组合交叉分配，产生差异化选课分布 |
| 修复版本 | v1.0.1 |
| 修复文件 | `src/kindergarten/util/InitDatabase.java` |

### Bug #3：每周食谱搭配不合理

| 字段 | 内容 |
|------|------|
| 问题描述 | 早餐只有1道主食（白米饭等），午餐只有1道荤菜，晚餐只有1道汤；菜品库中的水果（苹果、香蕉等）从未出现在食谱中 |
| 问题原因 | `InitDatabase.java` 第284~299行每餐只插入1道菜品，且未将水果类型（dish_type=5, id 17~20）纳入食谱 |
| 处理人 | 李瑞 |
| 修复方案 | 重新设计食谱结构：早餐=主食+水果，午餐=主食+荤菜+素菜+汤，晚餐=主食+素菜+汤；使用PreparedStatement参数化插入 |
| 修复版本 | v1.0.1 |
| 修复文件 | `src/kindergarten/util/InitDatabase.java` |

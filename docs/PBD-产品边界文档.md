## 1. 文档目的

本文档明确幼儿园管理系统的**系统边界、功能范围、排除项和约束条件**，防止项目范围蔓延，确保团队聚焦核心交付。

## 3. 功能范围矩阵

### 3.1 ✅ 系统范围内（In Scope）

| 模块 | 包含功能 | 边界说明 |
|------|---------|---------|
| 登录认证 | 登录、菜单权限区分、修改密码 | 仅2种角色：管理员/教师 |
| 幼儿学籍 | 增删改查幼儿信息、按班级查看 | 支持在园/离园状态管理 |
| 班级管理 | 查看9个预置班级、查看班级幼儿 | 班级不可增删，仅查看和管理 |
| 课程管理 | 动态增删课程、选课/退课、容量校验 | 每人限4门，课程有容量上限 |
| 食谱管理 | 菜品库CRUD + 每周排餐 | 按周一~周五，早中晚三餐 |
| 调班管理 | 任意调班、调班记录、容量校验 | 支持跨年级，记录操作日志 |
| 考勤管理 | 每日打卡、历史查看、出勤率统计 | 教师仅管理本班 |
| 数据统计 | 班级人数/课程选课/出勤率报表 | 控制台表格形式展示 |

### 3.2 ❌ 系统范围外（Out of Scope）

| 排除项 | 原因 |
|--------|------|
| 图形界面（GUI/Swing/JavaFX） | 题目明确要求仅控制台程序 |
| Web应用 / Spring框架 | 题目不允许JavaWeb |
| 家长端 / 家长查看功能 | 题目未要求，且超出控制台范围 |
| 学费/财务管理 | 不在需求范围内 |
| 成绩/评分管理 | 幼儿园阶段无成绩概念 |
| 教师排课/课表管理 | 题目关注幼儿侧课程分配 |
| 家园联系/通知功能 | 超出控制台程序边界 |
| 数据导入/导出（Excel等） | 非核心功能，时间有限 |
| 操作日志全记录 | 仅调班操作记录日志，其他不强制 |
| 数据备份/恢复 | 超出课程作业范围 |
| 多语言支持 | 仅中文 |

## 4. 用户边界

### 4.1 管理员操作边界

```
管理员可以：
  ✅ 登录/退出系统
  ✅ 增删改查幼儿信息
  ✅ 查看班级列表和班级幼儿
  ✅ 增删改查兴趣课程
  ✅ 管理菜品库
  ✅ 编辑每周食谱
  ✅ 执行调班操作
  ✅ 查看所有统计数据
  ✅ 修改自身密码

管理员不可以：
  ❌ 代替教师进行考勤打卡（教师专属）
  ❌ 删除预置班级
  ❌ 创建/删除其他管理员账号（系统预置）
```

### 4.2 教师操作边界

```
教师可以：
  ✅ 登录/退出系统
  ✅ 查看本班幼儿名单及详情
  ✅ 记录本班幼儿每日考勤
  ✅ 查看本班考勤历史
  ✅ 查看课程安排（自己负责的课程）
  ✅ 查看本周食谱
  ✅ 修改自身密码

教师不可以：
  ❌ 增删改幼儿信息
  ❌ 管理课程/食谱
  ❌ 执行调班操作
  ❌ 查看其他班级的数据
  ❌ 查看统计数据
```

## 5. 数据边界

### 5.1 数据量边界

| 数据类型 | 最大容量设计 | 说明 |
|---------|------------|------|
| 幼儿总数 | ≤ 150人 | 超出90人预设时仍可工作 |
| 单班人数 | ≤ 15人 | 默认10人，调班时校验上限 |
| 兴趣课程数 | ≤ 20门 | 管理员可动态添加 |
| 单课程人数 | ≤ 30人 | 默认15人，可配置 |
| 用户账号数 | ≤ 20个 | 管理员+教师 |
| 历史考勤 | 保留当前学期 | 跨学期数据不自动清理 |

### 5.2 数据生命周期

```
幼儿入园 → 录入学籍 → 分配班级 → 选择课程 → 日常考勤
         → [可能调班] → [可能退课/选课]
幼儿离园 → 状态标记为"离园" → 保留历史数据 → 不物理删除
```

## 6. 技术约束

| 约束项 | 具体要求 |
|--------|---------|
| 开发语言 | Java（JDK 8+） |
| 用户界面 | 仅控制台（System.in/System.out） |
| 数据库 | MySQL 8.x |
| 连接方式 | JDBC直连（不允许ORM框架） |
| 架构模式 | 分层架构，禁止跨层调用 |
| 第三方库 | 不允许使用Spring、MyBatis等框架 |
| 代码注释 | 每个类必须有：类说明、作者、日期、版本 |

## 7. 接口边界

### 7.1 层间调用规范

```
View 层 ──调用──▶ Service 层 ──调用──▶ DAO 层 ──操作──▶ MySQL
  │                  │                  │
  │ 只负责：         │ 只负责：         │ 只负责：
  │ - 显示菜单       │ - 业务校验       │ - SQL执行
  │ - 接收输入       │ - 数据组装       │ - 结果集映射
  │ - 格式化输出     │ - 事务管理       │ - 连接管理
  │                  │                  │
  │ 禁止：          │ 禁止：          │ 禁止：
  │ - 直接操作SQL    │ - 直接打印输出   │ - 业务逻辑判断
  │ - 业务逻辑判断   │ - 直接读取输入   │ - UI交互
```

## 8. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| MySQL连接失败 | 系统无法使用 | 启动时检测连接，友好提示配置信息 |
| 课程容量冲突 | 并发选课超员 | 选课时实时查询当前人数并校验 |
| 删除幼儿后数据残留 | 数据不一致 | 删除时级联清理选课和考勤记录 |
| 输入格式错误 | 程序异常 | 所有输入做格式校验，异常时重新提示输入 |
